Request / Response Editor

The request / response editor is used to create and edit the HTTP request used for the brute-force attack, and also to view responses being returned whilst scanning.

The HTTP request can be copied and pasted from a Man-in-the-middle proxy server log, and edited as seen fit. In cases where copied and pasted HTTP requests are used, the following points should be kept in mind:
  • The HTTP request should be adjusted to remove the protocol and host information from the request (ie: Instead of GET http://127.0.0.1/index.html use GET /index.html).
  • HTTP/1.1 can allow multiple HTTP requests and responses to use the same single connection. Set the HTTP Request Header Connection: to Close
  • Should the application being brute-forced be a virtually hosted site, ensure that the correct Host HTTP Request Header is set.
  • The strings "##1##" and "##2##" are used to indicate placeholders where parameters will be substituted for the brute force attack. Setting these fields will enable the Parameter 1 and Parameter 2 form fields which allow for the specification of parameters.
