Main GUI Components

1 - Request / Response Toggle
2 - Request / Response Editor
3 - Result List
4 - Parameter Toggle
5 - Parameter Editor
6 - Settings and Filters

1 - Request / Reponse Toggle

Tab control to display either the HTTP request or response in the Request / Response Editor (2).

2 - Request / Response Editory

Allows one to view and edit the HTTP request used in the brute force attack, or to view the HTTP responses as they are returned.

3 - Result List

The result list displays the results of the brute force attack, and consits of 4 columns.

These columns are as follows:

SCORE - The comparison score of the request compared to the Base Response. Values approaching 1 indicate the response was similar to the base response.
PARAMETER 1 - The value of Parameter 1 used in this request.
PARAMETER 2 - The value of Parameter 2 used in this request.
CONTENT - The content extracted from the "Content Extraction" components.

4 - Parameter Toggle

Tab control to display the settings for parameter 1 and parameter 2.

5 - Parameter Editor

Used to adjust the values of the parameters used in the brute-force attack.

6 - Settings and Filters

This component consists of a number of different pages. They can be briefly explained as follows:

Actions - Used to get the Base Response and also to Start, Stop or Pause brute force scans.
Target - Used to set target host, port, SSL settings and also network timeouts and retries.
Fuzzy Logic - Used to filter results based on specific scores.
Content Extraction - Content between the specified start and end tokens will be displayed in the result list.
